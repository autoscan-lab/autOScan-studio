# Product Overview

## Problem

Current TUI workflow is powerful but has friction for day-to-day grading:
- limited spatial context while reading student code
- policy/test management and code inspection are split mentally
- SSH/server workflow requires context switching

## Product Direction

`autOScan Studio` is a grading workbench:
- familiar editor-style layout
- autOScan tools always visible
- terminal and SSH built into the same environment

## Target User

- Teaching Assistants grading C labs with custom/manual judgment.

## Success Criteria

1. Faster per-submission grading time.
2. Less setup/context-switching overhead.
3. Higher confidence in final grade decisions.
