# UI Layout

## Main Regions

1. Left Sidebar: workspace files/folders.
2. Center Pane: code editor/viewer.
3. Right Sidebar: autOScan actions and results.
4. Bottom Panel: terminal/session output.

## Right Panel (autOScan)

Sections:
1. Policy selection
2. Run/scan controls
3. Compile/run result summary
4. Banned functions
5. Similarity/AI detection overview
6. Export actions

## UX Principles

1. Make manual grading primary, automation secondary.
2. Keep evidence near decisions (errors, output diff, matched spans).
3. Reduce hidden keyboard-only flows; support click-first usage.
