# Testing Strategy

## Goals

1. Prevent drift from `autOScan` behavior
2. Validate core TA workflows end-to-end

## Test Types

1. Contract tests for engine adapter
2. UI smoke tests for major flows
3. Snapshot tests for critical result panels
4. Manual QA checklist with real submission fixtures

## Exit Criteria (v1)

1. Equivalent grading outcomes to TUI on agreed fixture set
2. Stable SSH session handling
3. No critical crashes during grading loops
